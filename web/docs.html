08 Metadata Preservation

By default, pxlsmash strips EXIF metadata for smaller files. Use --keep-metadata to preserve GPS, camera info, orientation, and other EXIF data.

Terminal ‚Äî Keep metadata

$ pxlsmash ~/Photos/ --keep-metadata --quality 85

# Perfect for photographers ‚Äî preserve camera info + GPS

$ pxlsmash shoot/ --keep-metadata --smart-quality --output optimized/

11 CI/CD Integration

name: Optimize Images

on:

push:

paths: ['**/*.png', '**/*.jpg', '**/*.webp']

optimize:

- uses: actions/checkout@v4

12 Config File

16 Troubleshooting

"Metal unavailable" ‚Äî falls back to CPU ‚ñº

This is normal on Intel Macs or in CI/CD environments without a GPU. pxlsmash automatically falls back to vImage (Accelerate framework). Performance is still excellent ‚Äî just not GPU-accelerated.

WebP encoding fails on macOS 13 ‚ñº

Native WebP encoding requires macOS 14 (Sonoma) or later. On macOS 13, use --format jpeg or --format png instead.

AVIF/HEIC support not available ‚ñº

AVIF encoding requires macOS 14+. HEIC is supported on all macOS versions. Make sure you have Xcode Command Line Tools installed: xcode-select --install

"Permission denied" errors ‚ñº

Make sure the output directory is writable. pxlsmash checks permissions before processing. On macOS, you may need to grant Full Disk Access in System Settings ‚Üí Privacy & Security.

Large files (100MB+) processing ‚ñº

pxlsmash handles large files with per-file error handling. If a single file fails due to memory constraints, the rest of the batch continues. Use --verbose to see memory usage information.
